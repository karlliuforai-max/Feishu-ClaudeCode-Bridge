# v0.1.0 Agent 切换开发方案

> 状态：本方案已并入 v0.2.0 实现。v0.2.0 在此基础上完成 `/agent` 切换、Codex 后端、Agent 独立 session、旧状态迁移和文档更新。

## 目标

v0.1.0 支持同一个飞书机器人应用在运行时切换 Claude Code 和 Codex。用户不需要更换飞书 `app_id`、应用密钥、WebSocket 连接或消息权限，只需在当前飞书私聊或群聊里发送 `/agent` 命令，即可切换当前会话后续消息使用的 Agent。

用户体验需要尽量保持和现有 Claude Agent 一致：

- 私聊直接回复。
- 群聊仅在 @ 机器人时回复。
- 回复继续走 `reply_to(message_id, ...)`，确保落回消息来源会话。
- 附件仍下载到 `WORKDIR/.inbox`，再以本地路径交给当前 Agent。
- `session_scope`、同会话加锁串行、消息去重、启动闸门、卡片回复、图片抽取和兜底发送逻辑保持不变。

## 模型策略

Claude 侧只调整默认模型：

- Claude 默认模型改为 `claude-sonnet-4-6`。
- Claude 继续支持现有 `/model` 和 `[m:...]` 临时模型覆盖。
- Claude 继续支持现有短名：`opus`、`sonnet`、`haiku`、`fable`。

Codex 侧在 v0.1.0 固定为单模型：

- Codex 始终使用 `gpt-5.5`。
- `/model` 不切换 Codex 模型。
- `[m:...]` 不新增 Codex 模型别名。
- 切换到 Codex 时自动使用 `gpt-5.5`。
- 切回 Claude 时恢复 Claude 默认模型或当前 Claude 会话级 `/model` 覆盖。

确认记录：当前工作区执行 `codex debug models`，`codex-cli 0.140.0-alpha.2` 只列出 `gpt-5.5` 一个可见 Codex 模型。本地 OpenAI latest-model 参考也将 `gpt-5.5` 标为最新/默认模型，适用于 coding 和 tool-heavy workflows。通过 OpenAI docs helper 拉取官方 Codex manual 时，`https://developers.openai.com/codex/codex-manual.md` 返回 HTTP 403；因此 v0.1.0 实现以当前部署环境的 Codex CLI 模型目录作为准绳。

## 会话命令

`/agent`

查询当前飞书会话使用的 Agent、生效模型和可用切换命令。

`/agent claude`

把当前飞书会话切到 Claude Code。默认使用 `claude-sonnet-4-6`，除非该会话已有 Claude 侧 `/model` 覆盖。

`/agent codex`

把当前飞书会话切到 Codex。Codex 固定使用 `gpt-5.5`。

`/agent reset`

把当前飞书会话恢复到配置里的 `default_agent`。

`/new` 和 `/reset`

只重置当前激活 Agent 的 session。这样不会误清空另一个未激活 Agent 的上下文；之后切回另一个 Agent 时，它仍可续接自己的历史会话。

## 配置设计

保持旧配置兼容。现有 `config.json` 不修改也应该继续工作。

新增推荐字段：

```jsonc
{
  "default_agent": "claude",
  "model": "claude-sonnet-4-6",
  "codex_model": "gpt-5.5",
  "codex_sandbox": "workspace-write",
  "codex_approval": "never",
  "codex_profile": ""
}
```

兼容规则：

- `default_agent` 缺省时使用 `claude`。
- 顶层 `model` 缺省时使用 `claude-sonnet-4-6`。
- `codex_model` 缺省时使用 `gpt-5.5`。
- v0.1.0 中，如果 `codex_model` 不是 `gpt-5.5`，启动时直接给出清晰配置错误。
- 现有 `allowed_tools` 仍只作用于 Claude。

## 状态结构

`sessions.json` 从单 Agent 结构升级为多 Agent 结构。

目标结构：

```json
{
  "c:oc_xxx:u:ou_xxx": {
    "agent": "codex",
    "created_at": "2026-06-18 13:30:00",
    "chat_id": "oc_xxx",
    "chat_type": "group",
    "chat_name": "Example Group",
    "sessions": {
      "claude": {
        "sid": "claude-session-id"
      },
      "codex": {
        "sid": "codex-session-id"
      }
    },
    "models": {
      "claude": "claude-sonnet-4-6"
    }
  }
}
```

迁移规则：

- 旧字符串记录迁移为 Claude session。
- 旧对象里有顶层 `sid` 的记录迁移为 Claude session。
- 缺少 `agent` 时使用 `default_agent`。
- 缺少 `sessions` 时懒创建。
- 保留已有 chat 元数据。

## Agent 适配层

新增 Agent 中立入口：

```python
def ask_agent(key, text, chat_id="", chat_type="", text_delta_cb=None, model="") -> str:
    ...
```

Claude adapter：

- 复用现有 `claude -p` 实现。
- 保留现有 `stream-json` 渲染器和飞书流式卡片行为。
- 新 session 继续使用 `--session-id`，已有 session 继续使用 `--resume`。

Codex adapter：

- 新 session 使用 `codex exec`。
- 已有 session 使用 `codex exec resume <session_id>`。
- 固定传 `--model gpt-5.5`。
- 传 `--cd WORKDIR`。
- 传 `--sandbox <codex_sandbox>`。
- 传 `--ask-for-approval <codex_approval>`。
- 使用 `--output-last-message <tempfile>` 稳定捕获最终飞书回复。
- 在需要终端流或 session id 捕获时使用 `--json`。

Codex v0.1.0 流式策略：

- 如果 Codex JSONL 事件格式稳定，可先解析为终端进度日志。
- 如果无法稳定拿到文本增量，飞书 `stream_reply` 在 Codex 下自动降级为最终消息一次性回复。
- Claude 的流式体验保持不变。

## 实施步骤

1. 将 Claude-only 全局配置改造成 Agent-aware 配置，同时保留必要旧变量名以降低改动面。
2. 新增 `DEFAULT_AGENT`、`CODEX_MODEL`、Codex 沙箱和审批配置。
3. 用 Agent-aware session helper 替换 `_get_or_create_sid()`：
   - `_get_active_agent(key)`
   - `_set_active_agent(key, agent)`
   - `_get_or_create_agent_sid(key, agent, chat_id, chat_type)`
   - `_save_agent_sid(key, agent, sid)`
4. 保留 Claude 命令构造，新增 Codex 命令构造：
   - `_build_codex_new_cmd(text, model)`
   - `_build_codex_resume_cmd(text, sid, model)`
5. 增加 Codex 输出解析：
   - 从 `--output-last-message` 读取最终回复
   - 从 JSONL 事件中捕获 session id
   - stderr 兜底生成清晰错误
6. worker 日志从固定 `Claude 处理中` 改成当前 Agent 感知文本。
7. 在 `/model` 之前处理 `/agent` 命令。
8. 将 `/model` 改成 Agent 感知：
   - 当前 Agent 是 Claude 时，沿用现有行为，默认模型为 `claude-sonnet-4-6`。
   - 当前 Agent 是 Codex 时，提示 Codex v0.1.0 只使用 `gpt-5.5`，如需 Claude 模型切换请先 `/agent claude`。
9. 更新配置模板、README、CHANGELOG 和测试。

## 测试计划

新增或调整单元测试：

- Claude 默认模型是 `claude-sonnet-4-6`。
- Codex 模型固定为 `gpt-5.5`。
- 非 `gpt-5.5` 的 `codex_model` 会启动失败。
- `/agent` 查询能显示当前 Agent 和模型。
- `/agent codex` 只切换当前会话。
- `/agent claude` 能切回 Claude。
- 旧字符串 session 记录能迁移为 Claude session。
- 旧顶层 `sid` 对象能迁移为 Claude session。
- Claude 命令构造保持兼容。
- Codex 新会话命令包含 `codex exec`、`--model gpt-5.5`、`--cd`、sandbox、approval、JSON 输出和 output-last-message。
- Codex resume 命令包含 `codex exec resume <sid>`。
- Codex 状态下 `/model` 不修改 Codex 模型状态。

## 验收标准

- 同一个飞书机器人应用可同时服务 Claude 和 Codex 用户。
- `/agent codex` 后，同一飞书会话后续消息走 Codex。
- `/agent claude` 后，同一飞书会话后续消息走 Claude。
- Claude 默认模型是 `claude-sonnet-4-6`。
- Codex 始终运行 `gpt-5.5`。
- 旧 Claude-only 部署升级后继续可用。
- 旧 sessions 不丢失。
- 飞书路由仍然安全：回复只回原消息或原 chat。
- Codex 失败时返回用户可读的飞书消息，不静默丢消息。
